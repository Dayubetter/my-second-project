### 异常的作用
1. 定位bug
2. 作为方法内部的一种特殊返回值，以便通知上层调用者，方法的执行问题
```java
// 作为返回值
// return -1;
throw new RuntimeException("异常信息");
```
### 自定义异常
1. 自定义编译时异常
- 继承Exception
- 重写构造器
- 通过throw new 异常类(xxx)创建异常对象并抛出
> 特点：编译阶段就报错，提醒比较强烈 

ctrl+alt+t：创建try catch

### 异常的处理方案
1. 底层异常层层往上抛出，最外层捕获方法，记录下异常信息，并相应适应用户观看的信息进行提示
2. 最外层捕获异常后，尝试重新修复
