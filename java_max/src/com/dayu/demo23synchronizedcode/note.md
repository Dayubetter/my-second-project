### 线程同步
- 线程同步是线程安全问题的解决方案
- 让多个线程先后以此访问共享资源，这样就避免出现线程安全问题
- 线程同步的常见方案
    1. 加锁：每次只允许一个线程加锁，加锁后才能进入访问，访问完毕后自动解锁，然后其他线程才能再加锁进来
方法一：同步代码块
方法二：同步方法
方法三：lock锁

#### 同步代码块
作用：把访问共享资源的代码给上锁，以此保证线程安全
```java
synchronized(同步锁){
          访问共享资源的核心代码
        }
```
> 原理：每次只允许一个线程加锁后进入，执行完毕后自动解锁，其他线程才可以进来执行。
> 
> 建议使用共享资源作为锁对象，对于实例方法建议使用this，对于静态方法建议字节码（类名.class)对象作为锁对象

同步锁的注意事项
- 对于当前同时执行的线程来说，同步锁必须是同一把（同一个对象）
#### 同步方法
作用：把访问共享资源的核心方法给上锁，以此保证线程安全
```java
修饰符 synchronized 返回值类型 方法名(参数列表){
          访问共享资源核心代码
}
```
> 原理：每次只允许一个线程加锁后进入，执行完毕后自动解锁，其他线程才可以进来执行。
- 同步方法底层原理
  1. 同步方法其实底层也是有隐式锁对象的，只是锁的范围是整个方法代码。
  2. 如果方法是实例方法：同步方法默认用this作为的锁对象
  3. 如果方法是静态对象：同步方法默认用字节码对象.class作为锁对象

#### lock锁
Lock锁是JDK5开始提供的一个新的锁定操作，通过它可以创建出锁对象进行加锁和解锁，更灵活、更方便、更强大
Lock是接口，不能直接实例化，可以采用它的实现类ReentrantLock来构建Lock锁对象。

|构造器|描述|
|:---|:---|
|public ReentrantLock()|获得Lock锁的实现类对象|

|方法|描述|
|:---|:---|
|public void lock()|加锁|
|public void unlock()|解锁|
